博客好久没更新都积灰了，旧的主题已经审美疲劳了（主要其实也是历史遗留问题，因为原主题用的不是js不会调用第三方记录访客量），这回回来换个新皮肤作为新开始吧。本文记录一下换新过程中遇到的一些问题，供下次参考。

DevTools 无法加载 SourceMap

虽然说这个是警告，不管好像也没有什么问题？但事实上不是这样，这个主题加载了一个第三方的组件库（MDUI），而找不到这个map映射文件的话，就会让博客变得非常丑。就像下面这样：

解决方案

到MDUI的GitHub仓库下mdui/dist/js和mdui/dist/css下载相应的 .map文件放到本地主题相应目录下即可。

https协议下无法访问http资源

解决方案

在本地主题目录下/layout/_partial/head.ejs文件中\<head>中加上\<meta http-equiv =”Content-Security-Policy” content=”upgrade-insecure-requests”>

git相关

博客部署时遇到Please make sure you have the correct access rights and the repository exists.

解决方案

  • 到用户目录下.ssh文件夹内删除known_hosts文件

  • 重新生成私钥公钥ssh-keygen -t rsa -C "840841939@qq.com"

  • 将id_isa.pub文本内容填到GitHub上

小知识

Windows下用命令行以文本内容查找文件findstr /s /i /n "要查找的内容" *.*

/s：在当前文件夹和子文件夹下查找

/i：不区分大小写

/n：显示行号

*.*：要查找的文件的文件名格式

惊了！写完这篇部署上传的时候有遇到了问题，连夜打开继续记录

上传文件大小限制，将限制改大即可git config http.postBuffer 52428800
